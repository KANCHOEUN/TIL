# 03. 상수 ( Constants )



1. **변수를 상수화** : 변수에 값을 딱 한 번만 할당할 수 있음

   ```java
   final int MAX_SIZE = 100;
   ```

   final 키워드에 의해 MAX_SIZE 라는 변수가 가지고 있는 값은 변경이 불가능. 상수화가 된 것

   final 키워드가 붙은 변수를 초기화를 하지 않았다면 딱 한 번만 초기화 가능.



2. **리터럴** : 자료형을 기반으로 표현이 되는 상수

   리터럴을 보면 값 뿐만 아니라 데이터 타입을 알아야 한다 !



> ###### long형 리터럴의 표현 방법
>
> ```java
> System.out.println(3147483647 + 3147483648); // ERROR : Integer Number Too Large
> System.out.println(3147483647L + 3147483648L); // l 또는 L 을 붙여 long형 리터럴로 표현
> ```
>
> 모든 정수 리터럴의 기본은 int형
>
> BUT int 자료형으로 표현할 수 있는 값의 범위를 넘어섰음



> ##### 상수와 리터럴 둘 다, 변하지 않는 값 ( 데이터 ) 을 의미
>
> **상수(Constants)** 는 변하지 않는 변수
>
> ex. C, C++ 에서는 const / Java 는 final 제어자를 사용
>
> ```java
> final Test t1 = new Test();
> t1 = new Test(); // 불가능
> t1.num = 10; // 가능
> ```
>
> 참조 변수를 상수로 지정하는 경우,
>
> 참조 변수가 상수, 즉 메모리의 주소값이 변하지 않는다는 의미이고
>
> 그 주소가 가리키는 데이터들까지 상수라는 의미는 아님
>
> 
>
> **리터럴(Literal)** 은 데이터 그 자체를 뜻함
>
> 변수에 넣는 변하지 않는 데이터를 의미
>
> ```java
> int a = 1;
> ```
>
> int 앞에 final 을 붙일 시, a는 상수가 됨
>
> 여기서 리터럴은 1 을 뜻함
>
> 
>
> 따라서 상수는 변하지 않는 변수를 의미 ( 메모리 위치 )
>
> 리터럴은 변수의 값이 변하지 않는 데이터 ( 메모리 위치 안의 값 ) 을 의미



##### 정수형 상수의 이진수 표현 방법

: 0B 또는 0b 를 붙여서 이진수 표현

```java
byte seven = 0B111;
int num205 = 0B11001101;
```



##### 언더바 삽입

: 원하는 위치에 언더바 삽입함으로써 가독성을 높일 수 있음

```java
int num = 100_000_000;
int num = 12_34_56_78_90;
```
